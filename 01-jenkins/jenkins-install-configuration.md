## Установка Jenkins

Приложение Jenkins распространяется в виде war файла и запускается командой java -jar jenkins.

1. Скачайте War пакет  по ссылке https://www.jenkins.io/download/ 

2. Переместите файл jenkins.war в папку c:\tools\jenkins.

3. Откройте терминал в этой директории и выполните команду

```
java -jar jenkins.war
```

Внимательно посмотрите в консоль и найдите строчку с временным паролем.

Не закрывайте терминал до конца работы c jenkins. 

Откройте браузер и перейдите в по ссылке http://localhost:8080/ 
На первой странице необходимо ввести временный пароль, чтобы перейти в настройки окружения Jenkins.

Jenkins предлагает выбрать необходимые плагины. Выберите пункт с плагинами по умолчанию.

Последнее окно будет создания администратора.

Рабочее окно программы выглядит следующим образом:

Создание конфигурации

Выберите пункт "Создать Item". Item - это ваш процесс описанный выше.

Дайте название проекту CheckDev и выберите пункт "Создать задачу со свободной конфигурацией".


В нашем случае большая часть действий ложиться на Maven.

Вначале необходимо указать расположение проекта - Github.

Укажите ветку main.

Далее нужно добавить выполнение Maven сборки.

Проект CheckDev состоит из микросервисов. Микросервис - это автономное приложение по аналогии с dreamsjob или accidents.

Сохраните настройки. Теперь можно запустить выполнение сценариев. Кликните кнопку "Собрать проект".






Внизу появится история запусков. Если кликнуть на один из пунктов, то можно посмотреть лог выполнения.
